# agentMemory

Workflow-aware retrieval engine for [maestro](https://github.com/ReinaMacCredy/maestro) memory files.

Maestro writes memories as `.md` files. agentMemory indexes them and provides hybrid search with keyword, semantic, and workflow-specific signals that no general-purpose memory system has.

## How It Works

```
.maestro/
  features/auth/memory/decision-jwt.md    maestro writes these
  features/auth/memory/exec-setup-db.md
  memory/global-convention.md
  retrieval-index.json                    agentMemory writes this (index only)
  feedback.jsonl                          agentMemory writes this (learning data)
```

agentMemory never touches `.md` files. It only reads them, builds an index, and returns ranked results.

## Retrieval Signals

Six signals, weighted for workflow dominance:

| Signal | Weight | What It Does |
|---|---|---|
| Semantic | 0.25 | Cosine similarity via ONNX embeddings (384-dim) |
| Keyword | 0.15 | BM25-style TF-IDF from pre-extracted tokens |
| Pipeline Stage | 0.20 | Boosts memories matching current workflow stage |
| Dependency Graph | 0.20 | BFS walk through task deps -- upstream memories score higher |
| Execution Feedback | 0.15 | Memories correlated with successful tasks get boosted |
| Recency | 0.05 | Newer memories score slightly higher |

Workflow signals (stage + graph + feedback) are 0.55 combined. This is the core differentiator vs general memory systems like Mem0 or Supermemory.

## MCP Tools

| Tool | Purpose |
|---|---|
| `memory_query` | Hybrid search with all 6 signals |
| `memory_compile` | Budget-aware context assembly (token-capped, MMR-diverse) |
| `memory_feedback` | Record task outcomes for the learning loop |
| `memory_graph` | Traverse task dependencies for upstream memories |
| `memory_admin` | Reindex, stats, health check |

## Setup

```bash
bun install
```

### Standalone MCP Server

```bash
MAESTRO_DIR=/path/to/.maestro bun run dev
```

### As Maestro Dependency

In the maestro repo:

```bash
bun add file:../agentMemory
```

Maestro's toolbox adapter auto-detects agentMemory and uses it for enhanced DCP memory selection. When not installed, maestro falls back to standard keyword scoring.

## Sidecar Index

`retrieval-index.json` caches:
- SHA-256 checksums (detect stale entries)
- Keyword tokens (extracted from body + tags + filename)
- Embedding vectors (384-dim, generated by ONNX when available)
- Metadata snapshot (tags, category, stage, priority)

If deleted, it rebuilds automatically on next query. Incremental sync only re-indexes changed files (checksum comparison).

## Architecture

```
src/
  store/           Scanner, index manager, frontmatter parser, keywords
  retrieval/       Hybrid merge, keyword BM25, semantic cosine, MMR diversity
  workflow/        Pipeline stage scoring, dependency graph walk, execution feedback
  embeddings/      ONNX provider (all-MiniLM-L6-v2, optional)
  pipeline/        Semantic chunking
  server/          MCP server + 5 tool handlers
```
